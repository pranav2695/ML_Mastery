# Gradient descent
It is the first order iterative optimization algorithm for finding the local minima of a diffrentiable function. The idea is to take the repeatedly steps in the opposite direction of the gradient of the function at the current point because this is the direction of the steepest descent   
It is used in Linear Regression, Logistic Regression and complete Deep learning is based on Gradient Descent.

### When to stop 
if b_new - b_old = very small number it means we are converging, we can limit the number of epochs or we can check the difference 
if we plot the loss with no of epochs then also we can find when to stop the gradient descent as loss will not change after increasing the no of epochs  
## Learning Rate 
- we need to tune this values based on the data only, we need to run this and then find how many epochs are required on the particular data.
- we have staddle point issue also: it will take lot of time because it is very flat
- we have feature are in the same scale it will take less time, so always we should scale it first 
## Different types of Gradient descent
1. Batch GB: we take all the data to do the update, it is used when we have convex function and data is small.
2. Stochastic GD: we take only one data point to do the update
3. Mini batch GD: we define the batch size like 30 , after 30 data point to do the update
