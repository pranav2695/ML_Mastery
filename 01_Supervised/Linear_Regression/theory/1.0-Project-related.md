## Pipeline
Why is this better than manual coding?
If an interviewer asks, "Why bother with a Pipeline?", give these three reasons:

Prevents Data Leakage: When you call .fit(), the pipeline learns the median and mean only from the training data, then applies those same values to the test data. Manual scaling often leads to accidentally "peeking" at the test data's mean.

Atomicity: You can save this full_pipeline object as a file (using pickle or joblib). When you deploy your model to production, you just call pipeline.transform(new_data), and all cleaning happens automatically.

Hyperparameter Tuning: You can use this pipeline inside a GridSearchCV to test if the "median" strategy is actually better than a "mean" strategy for your imputer.


Input: Raw DataFrame with mixed types and missing values.

Split: ColumnTransformer sends numbers left and text right.

Process: Numbers get filled and scaled; Text gets encoded.

Merge: Both paths are concatenated back together into a single NumPy array.

Output: A clean, purely numerical matrix ready for a Machine Learning algorithm.
## Different things we can do in pipeline
1. Feature Engineering (Custom Transformers)You aren't limited to built-in functions. You can create your own logic (e.g., calculating "rooms per household" from "total rooms" and "households").FunctionTransformer: Great for simple, stateless transformations like taking the log of a column to fix skewed data.Custom Classes: You can write a class that inherits from BaseEstimator and TransformerMixin to create complex features that "learn" from the data.
2. Feature SelectionIf you have 100 features but only want the 10 most important ones to speed up the model and reduce noise.SelectKBest: Picks the top $K$ features based on statistical tests (like Correlation or Chi-Square).VarianceThreshold: Removes features that don't change much (e.g., a column where 99% of the values are the same).
3. Dimensionality ReductionWhen you have too many features (the "Curse of Dimensionality"), you can compress them.PCA (Principal Component Analysis): Combines multiple correlated features into a smaller set of "Principal Components" that capture most of the variance.
4. Handling Skewed DataMany models (like Linear Regression) perform better if the data is normally distributed (bell curve).PowerTransformer: Applies transformations like Box-Cox or Yeo-Johnson to make the data more Gaussian-like.QuantileTransformer: Forces the data into a specific distribution (like Uniform or Normal), which is very robust against outliers.
5. Advanced Categorical EncodingBeyond OneHotEncoder, there are other ways to handle text data:OrdinalEncoder: For categories with a natural order (e.g., "Low", "Medium", "High" $\rightarrow$ 1, 2, 3).TargetEncoder (from the category_encoders library): Replaces a category with the average target value for that category. Extremely powerful for high-cardinality features like "Zip Codes."
6. The "Final Boss": The Model ItselfYou can include the Machine Learning model (e.g., LinearRegression or RandomForest) as the final step of the pipeline.  

full_pipeline_with_model = Pipeline([
    ("preprocessor", full_pipeline),  
    ("regressor", LinearRegression())
])

Type,Example Component,Purpose
